    Introduction

    Utilize a documentação aqui descrita para conhecer melhor os endpoints da Jobzi API. Com estes endpoints você pode manipular toda a informação básica para o funcionamento do jobzi.com como: usuários, autenticação de usuários, curriculo, vagas, sugestões, recomendações, etc.

    Se você é desenvolvedor, sinta-se à vontade para corrigir ou atualizar esta documentação. Para isto basta clonar o Código do bitbucket e modificar ou criar documentos “.md” no diretório “/source/includes”. Para mais exmplos de manipulação e edição, utilize a documentação oficial do Slate.

    Ao criar novos arquivos “.md”, não esqueça de incluir uma referência no arquivo “index.md” na raiz do projeto.

    Delete User

    DELETE /v2/users/<ID> HTTP/1.1
Accept: application/json
Host: api.jobzi.com, apidev.jobzi.com, apistage.jobzi.com

    HTTP/1.1 201 Created
Content-Type: application/json

{
"message": "User successfully disabled."
}

    Exclui um usuário cadastrado.

    URL Parameters

    Parâmetro Descição
    ID O ID do usuário a ser excluído
    Os dados do usuário não são efetivamente excluídos. Apenas um flag no objeto usuário é modificado: “deleted” = true.

    Errors

    Todo erro retornado pela API tem o formado do seguinte objeto: {message: "alguma mensagem relacionada"}

    Jobzi API usa os erros de código listados abaixo.

    Erro (cod.) Significado
    400 Bad Request – A requisição não foi entendida.
    401 Unauthorized – API Key errada ou JWT inválido no Authorization header.
    404 Not Found – Um resource específico não pode ser achado.
    409 Conflict – A requisição foi parcialemente entendida.
    500 Internal Server Error – Erro interno do servidor, causado em geral por erros provenientes de conecções com outras máquinas (ex.: Banco de dados).
